# Start Here

## I Just Want To Use The App

If you can see `RUNME - Start TRANSCRIBER.cmd`, double-click that. It installs the app on first run, then opens TRANSCRIBER at localhost.

If this folder came from a release ZIP:

1. Double-click `1 Install TRANSCRIBER.cmd`.
2. Wait for it to finish.
3. Double-click `2 Start TRANSCRIBER.cmd` or the Desktop shortcut named `TRANSCRIBER`.

If this is the source project folder:

1. Double-click `Setup.cmd`.
2. Wait for it to finish.
3. Double-click the Desktop shortcut named `TRANSCRIBER`.

After that, you normally do not need to open the project folder again.

## Useful Files

- `Setup.cmd` - first-time install
- `Launch.cmd` - start the app on this computer
- `Launch-LAN.cmd` - start the app for same Wi-Fi phone/tablet access
- `Setup-Speakers.cmd` - optional speaker labels
- `README.md` - full usage notes
- `scripts\package_windows.ps1` - builds a clean Windows release ZIP

## Files You Can Ignore

- `.venv` - installed Python dependencies
- `tools` - bundled FFmpeg tools
- `transcriber` - app source code
- `assets` - icons and theme images
- `scripts` - maintenance scripts
- `.github` - GitHub automation

These folders are part of the app. You do not need to edit them to use TRANSCRIBER.
